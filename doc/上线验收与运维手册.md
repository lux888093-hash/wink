# 红酒扫码小程序上线验收与运维手册

- 更新日期：2026-04-16
- 适用范围：阶段 5 上线准备、灰度发布、运维巡检

## 1. 上线前环境变量

生产环境至少需要配置：

1. `APP_ENV=production`
2. `MINIPROGRAM_BASE_URL=https://你的正式域名`
3. `CORS_ALLOWED_ORIGINS=https://你的正式域名`
4. `DATABASE_URL` 或 `PGHOST/PGDATABASE/PGUSER/PGPASSWORD`
5. `ADMIN_SESSION_PEPPER`
6. `ADMIN_BOOTSTRAP_PASSWORD`
7. `MINIAPP_SESSION_SECRET`
8. `WECHAT_APPID`
9. `WECHAT_APPSECRET`
10. `WECHATPAY_MCHID`
11. `WECHATPAY_MCH_SERIAL_NO`
12. `WECHATPAY_PRIVATE_KEY_PATH`
13. `WECHATPAY_NOTIFY_URL`
14. `WECHATPAY_API_V3_KEY`
15. `WECHATPAY_PLATFORM_PUBLIC_KEY_PATH` 或 `WECHATPAY_PLATFORM_CERT_PATH`
16. `ENABLE_DEV_RESET=false`
17. `BACKUP_DIR`

建议配置：

1. `REDIS_URL`
2. `OBJECT_STORAGE_BASE_URL`
3. `CDN_BASE_URL`
4. `LOG_DIR`

## 2. 上线前检查

启动服务后执行：

```powershell
Invoke-RestMethod http://127.0.0.1:3100/api/health
Invoke-RestMethod http://127.0.0.1:3100/api/health/readiness
Invoke-RestMethod http://127.0.0.1:3100/api/health/metrics
```

验收要求：

1. `/api/health` 返回 `ok=true`。
2. `/api/health/readiness` 在生产环境返回 HTTP 200 且 `ready=true`。
3. `warnings` 为空，或只剩已确认可接受的非阻塞提示。
4. `capabilities.wechatLogin=true`。
5. `capabilities.wechatPay=true`。

## 4. 核心验收流程

### 4.1 提取码体验

1. 后台生成提取码批次并绑定酒款、歌曲。
2. 导出 CSV。
3. 小程序输入有效提取码。
4. 专属页只展示绑定酒款和绑定歌曲。
5. 同一提取码二次输入被拒绝。

### 4.2 商城交易

1. 小程序添加商品到购物袋。
2. 订单确认页填写收货地址。
3. 创建订单后库存进入预占。
4. 支付成功后库存扣减。
5. 后台录入物流公司和物流单号。
6. 用户申请退款。
7. 后台确认退款并按需入库。

### 4.3 会员与数字权益

1. 用户开通会员，先生成待支付订单。
2. 支付成功后才发放会员有效期。
3. 未过期会员续费时，到期日叠加。
4. 单曲解锁先生成待支付订单。
5. 支付成功后才发放下载权益。
6. 下载链接短时有效，使用后记录下载日志。

## 5. 灰度与回滚

建议流程：

1. 先在测试小程序和测试商户号完成真机支付。
2. 生产发布前执行 `npm test` 和 `npm run backup`。
3. 首批只投放少量提取码和少量商城商品。
4. 观察 `/api/health/metrics`、后台订单、支付回调、下载日志。
5. 若出现支付或订单异常，先关闭商品上架，再回滚服务版本。
6. 回滚前后都保留 `BACKUP_DIR` 中的快照。
